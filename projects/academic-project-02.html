Project Case Study

Geometric Modeling

Bezier Curves/Surfaces & Triangle Meshes with Half-Edge Data Structures

Timeline

2 weeks (July 2025)

Project Type

Class Project

Course

CS 184: Foundations of Computer Graphics

Team Size

Solo

Context

The 2nd project of this course challenged us to explore topics on geometric modeling that were covered in lecture. We were to build Bezier curves and surfaces using De Casteljau’s algorithm, manipulate triangle meshes represented by half-edge data structure, and implement loop subdivision. Parts 3 to 6 were more difficult to implement correctly since they involved managing pointers, which were exposed to us as iterators in the project.

UC Berkeley’s CS 184 Project #2

For this project, starting with Part 1, I learned how the de Casteljau algorithm evaluates Bézier curves through repeated linear interpolation between adjacent control points until only one final point remains. Implementing this step-by-step made the recursive geometric meaning of Bézier curves much clearer, especially when visualizing each intermediate layer. In Part 2, I extended this idea to Bézier surfaces by applying the 1D de Casteljau algorithm twice: first across each row of the control point grid and then again down the resulting column of intermediate points. In Part 3, I implemented area-weighted vertex normals, which helped me understand how smooth shading is produced by averaging the normals of all faces surrounding a vertex. This made the difference between flat shading and Phong shading very clear visually. In Part 4, I implemented edge flipping using the half-edge data structure, which required carefully updating all affected half-edges, vertices, edges, and faces. I did run into an issue regarding my pointers as when I initially flipped some of the edges, a triangle across the operated edge would seem to disappear. After rescanning and retracing each connection on my Half-edge table, I noticed I had set the boundary half-edge c -> d as the pointer for one of the inner triangle faces, when the correct one was half-edge d -> c. In Part 5, I implemented edge splitting by inserting a midpoint vertex, creating new half-edges, edges, and faces, and carefully reconnecting the local mesh structure. This part was built directly on edge flipping and showed how half-edge tables are useful for tracking complex pointer changes. Finally, in Part 6, I implemented Loop subdivision for mesh upsampling by computing new vertex positions, splitting original edges, flipping certain new edges, and updating all vertex positions. This demonstrated how subdivision increases mesh resolution and smooths surfaces, while also showing that simple shapes like cubes can become asymmetric if the mesh connectivity or preprocessing is not ideal.
