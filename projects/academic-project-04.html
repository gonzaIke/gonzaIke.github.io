Project Case Study

Ray Marching Simulation

SDF-Based Ray Marching, Volume Rendering, and Collision Detection

Timeline

3 weeks (August 2025)

Project Type

Class Project

Course

CS 184: Foundations of Computer Graphics

Team Size

4 members Isaac Gonzalez Buqi Liao Alan Wang Helin Zhou

Context

Given the tight schedule of the course during the summer as it is lauded as one of the toughest in the EECS department, our project’s timeline is shorter than the 6 weeks usually given, meaning that the scale of our ideas had to be limited. Nonetheless, the team frequently discussed project ideas and methodologies with the professor to choose a graphics problem that we wanted to research and solve with an executable programming project that was exciting to take on and add to our portfolios.

UC Berkeley’s CS 184 Final Project

Our project's focus here is with Signed Distance Fields (SDFs) and Ray Marching. The main problem we wanted to solve was that traditional collision detection often depends on complex mesh representations, can become unstable under high acceleration, and is usually handled separately from volume rendering. To address this, we used SDFs as a shared representation for both rendering and collision detection, which allowed the same scene description to support visual rendering and physical interaction. We implemented a ray marching rendering shader that could render SDF-based objects in real time while also using the distance field information for collision response. For the basic scene, we compared normal gravity and 100x gravity to test how stable the collision methods were under more extreme conditions. Under normal gravity, discrete collision detection, continuous collision detection, and ray marching all performed reasonably well, but under 100x gravity, the normal discrete method became much less reliable. CCD and ray marching were more stable in high-acceleration cases, with ray marching being especially useful because it could adaptively step through the SDF and detect collisions more robustly. We also implemented frictional collision response so the cloth points could react more realistically when interacting with the SDF objects. After testing the single-object setup, we moved toward a dynamic scene with two moving SDF volumes instead of using the previous static sphere mesh. This included a moving sphere and a moving cube, where the full scene SDF was computed by taking the minimum distance between the individual object fields. We visualized the cloth using a point cloud representation and observed that it still collided accurately with both moving objects in real time. This showed that the SDF volume method could handle more complex dynamic geometry while still remaining efficient enough for interactive simulation. Overall, the project helped us better understand physics simulation, collision stability, and the tradeoff between visual quality and performance, and in the end, we successfully connected ray marching, SDF volume rendering, dynamic scene processing, and multiple collision detection methods into one real-time simulation pipeline.
