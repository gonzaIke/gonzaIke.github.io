DIAS (Dual Interface Arm System)

Spacesuit Display (SSD) for Lunar Surface EVAs

Context

NASA’s Joint AR team supports the development of augmented reality systems and visual informatics tools for human spaceflight operations at Johnson Space Center. As NASA prepares for the Artemis lunar surface missions, the team continues to advance suit-integrated display concepts that can improve astronaut navigation, task execution, communication, and EVA situational awareness. For this NASA SUITS Challenge, teams must explore how displays, voice assistants, AI integration, and interoperability between spacesuits and rovers can support future EVA operations, where crewmembers must navigate, monitor telemetry, complete mission tasks, and coordinate with connected lunar assets.

Objective

Design and develop a spacesuit display system and AI voice assistant to support one EVA crewmember during a simulated lunar surface mission. The interface needed to present mission-critical telemetry, reduce cognitive load, and help the crewmember complete EVA tasks efficiently through concise visual and voice-based interactions. The system also required guardrails for AI responses to ensure reliability in mission-critical scenarios.

Overview

The Spacesuit Display (SSD) is a dual wrist-mounted display system implemented through the Dual Interface Arm System (DIAS) concept. Designed for a simulated Artemis lunar EVA scenario, the SSD proposes a hands-free interface that would help an EVA crewmember navigate, monitor telemetry, complete procedures, and coordinate with connected lunar assets without relying on a handheld device. The concept uses two phone-based displays attached to the astronaut’s wrists, allowing information to remain accessible across the user’s field of view while reducing the need for repeated manual interaction.

The first display, called the Mission Interface, would focus on navigation, map awareness, and task progression. It would feature a 1:1 2D lunar map, a legend sidebar, path indicators, safe-travel warnings, airlock return guidance, and tools for placing, removing, or navigating to custom pins. The legend sidebar would allow the crewmember to select which assets, paths, pins, or mission markers appear on the map so the display can stay focused on task-relevant information. The Mission Interface would also support procedure lists and task progression, allowing the astronaut to advance EVA steps through either voice commands or manual button interaction.

The second display, called the Telemetry Monitor, would organize spacesuit and EVA data from the Telemetry Stream Server into readable groups such as suit status, DCU values, health indicators, LTV proximity data, warnings, and task-board information. Its widget-based layout would allow the astronaut to rearrange telemetry groups, resize text, hide lower-priority values, or prioritize mission-specific data based on personal preference or EVA context. Even when certain values are hidden from view, the AI voice assistant would continue monitoring telemetry and alert the crewmember to abnormal statuses through tiered warnings, subtitles, and voice feedback.

The SSD concept also includes an AI Virtual Compartment that would support concise voice-based interactions for navigation, procedure retrieval, telemetry checks, settings changes, and location-based notifications. During setup, the astronaut would designate which wrist device serves as the Mission Interface and which serves as the Telemetry Monitor, then adjust brightness, audio, widget layout, and input preferences before egress. Although the displays could be placed according to user preference, the concept anticipates that the Mission Interface may be worn on the non-dominant arm so the astronaut can interact manually when voice control is unavailable or undesirable. By combining voice control with physical button options, the system avoids forcing the crewmember into a single input method during high workload EVA tasks.

For human-in-the-loop (HITL) testing, the SSD concept would evaluate whether a dual asymmetrical wrist-mounted UI can help users complete EVA tasks faster and with fewer navigation errors than a single-screen or handheld interface. We would also examine how interface type, cue priority, cue transparency, and input method affect task completion time, telemetry retrieval, field-of-view obstruction, and perceived efficiency during simulated lunar surface operations.
